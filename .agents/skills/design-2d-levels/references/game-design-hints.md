# 2Dゲームデザインとレベルデザインの原則

## 目次

1. 目指す体験から逆算する
2. 行動と結果の因果を明確にする
3. 難易度を精密操作だけに依存させない
4. レベルでルールを教える
5. 新要素を安全に試させる
6. 判断のない待ち時間を減らす
7. ゲームフィールで理解を支える
8. 観察型プレイテストを行う

## 1. 目指す体験から逆算する

ゲームを Mechanics、Dynamics、Aesthetics に分けて考える。

- **Mechanics**: 操作、物理、得点、勝敗などのルール。
- **Dynamics**: ルールの組み合わせから生じる行動や戦略。
- **Aesthetics**: 挑戦、発見、達成感、表現などの体験。

機能を増やす前に、どの感覚を中心にするかを言語化する。
実装したルールだけでなく、実際に生じているプレイを観察する。

## 2. 行動と結果の因果を明確にする

現実性よりも、一貫性、予測可能性、失敗理由の理解を優先する。
補助表示は答えではなく、仮説を立て結果から修正するための足場にする。

## 3. 難易度を精密操作だけに依存させない

- 安全だが時間や資源のかかる方法を置く。
- 難しいが大きな報酬を得られる方法を置く。
- 複数の能力や理解の仕方で突破できるようにする。
- クリア条件と上級者向け目標を分ける。

速度上昇、判定縮小、時間短縮だけで難しくしない。

## 4. レベルでルールを教える

1. **導入**: 新要素を安全かつ単純に見せる。
2. **展開**: 少し難しい条件で使わせる。
3. **転換**: 反転、複合、意外な使い方を求める。
4. **結論**: 学習内容を短い試験として使わせる。

この構造は単一レベルにも、複数レベルの章構成にも使える。

## 5. 新要素を安全に試させる

- 新要素を二つ以上同時に初登場させない。
- 最初の使用時は失敗の代償を小さくする。
- 一度成功させてから既知要素と組み合わせる。
- 失敗時に、どの理解が違っていたかを読み取れるようにする。

複雑さは要素数ではなく、既知要素の組み合わせから作る。

## 6. 判断のない待ち時間を減らす

予測、緊張、計画を生む待ち時間には価値がある。
新しい判断や情報がない時間は短くする。

- 失敗から再挑戦までを短くする。
- 待機中にも次の判断材料を見せる。
- 演出時間を成功の重要度に比例させる。
- 可能な場面ではプレイヤー入力で展開を進める。

## 7. ゲームフィールで理解を支える

- 入力には即時かつ一貫した反応を返す。
- 小さな成功と大きな成功を演出で区別する。
- 音、動き、色、時間変化を同じ意味へ揃える。
- 失敗にも次の行動を考えられる情報を残す。
- 強い演出を常用せず、重要な瞬間のために差を残す。

## 8. 観察型プレイテストを行う

感想だけでなく、実際の行動を記録する。

- 最初の行動までに何秒かかったか。
- 同じ失敗を何回繰り返したか。
- 失敗後に行動を修正したか。
- 意図した仕組みを自力で発見したか。
- クリア後も別解や高得点を試したか。

攻略を先に説明せず、プレイ後に「何を狙ったか」「なぜ失敗したと思うか」
「次に何を変えたか」を聞く。設計意図と回答が違う場合は、
プレイヤーより先に情報提示とレベル構造を疑う。

比較テストでは一度に複数条件を変えず、問いを一つに絞る。
